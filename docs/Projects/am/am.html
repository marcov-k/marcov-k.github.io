GitHub Repository

Fun Fact

Although the simulation is written in C++, it is rendered through the C rendering library Raylib rather than its C++ counterpart due to not requiring any complex object-oriented functionality for the graphics themselves.

Overview

Auto-Moeba is a particle life simulation loosely based on Conway's Game of Life and inspired by various other similar simulations I have come across. My primary purpose for developing the project is to become more familiar with C++, particularly its inheritance and polymorphism systems, which I did not need to use in my previous project involving the language - Neural Network Notions. The simulation includes a variety of different particle types which interact in unique ways with each other. Although Auto-Moeba is relatively simple, it is still capable of producing interesting and complex behaviours given the right conditions.

Particle movement is calculated using asymmetric Newtonian-style physics interactions. Each particle attracts/repels every particle type by a given amount, which is then used in place of acceleration for computing the motion of every particle using standard kinematics. Whenever a particle creates an offspring, the attraction values of the new particle are randomly mutated, leading to changing behaviours over time. A saving/loading system is also implemented using a custom .moeba file format, enabling any unique and interesting setups to be revisited at a later time. A more detailed description of both the particle types currently present in the simulation and the .moeba file format can be found on the Auto-Moeba GitHub repository.

Fun Fact

This is my first project to handle its file management through the standard Windows API as opposed to implementing its own more limited system, which allows the user to save and load the simulation's state to and from any location on the hard drive.
